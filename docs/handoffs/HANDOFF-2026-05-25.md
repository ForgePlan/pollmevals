## ✅ Best practices (validated в этой сессии)

### 1. Library-first rule — MANDATORY (CLAUDE.md § Library-first)

**Перед любым "implement X":**
1. `mcp__context7__resolve-library-id(libraryName, query)` — найти library
2. `mcp__context7__query-docs(libraryId, question)` — API + version
3. Pin version в pyproject.toml / package.json
4. Wrap library под Protocol seam, не replace
5. Custom code ТОЛЬКО когда library absent/abandoned/license-incompat

**Каноничный stack для POLLMEVALS** (см. NOTE-004 Section 5):
- Cyclomatic: `radon` (Py) / `lizard` (multi) / `eslint --max-complexity` (TS)
- Coverage: `coverage.py` / `c8` / `vitest --coverage`
- Lint: `ruff` / `eslint` / `clippy`
- Type: `mypy --strict` / `tsc --strict`
- Security: `bandit`, `semgrep`, `trivy`, `pip-audit`, `npm audit`, `gitleaks`
- Docstring: `interrogate` / `jsdoc-coverage`
- Profile: `cProfile` + `pyinstrument` / `clinic.js`
- α (Krippendorff): `krippendorff` PyPI package — `level_of_measurement="ordinal"`, `reliability_data=` requires **rows=judges, cols=items** (НЕ inverted)
- Retry: `tenacity` (Py) / `p-retry` (Node) — НЕ catch+sleep loop
- HTTP: `httpx` (already in use) / `axios`/`undici`
- Validation: `pydantic` v2 (already) / `zod`
- LLM eval: Inspect AI (already, EVID-004)
- Cost: `litellm.cost_per_token(model, prompt_tokens, completion_tokens)` — Library-first applied

### 2. Parallel execution — file ownership boundaries

**Pattern**: 3 ortho file ownership tracks одним сообщением (parallel `Agent({...})` calls).

```
🔵 Track A → apps/eval-core-py/src/orchestrator/X.py + tests/test_X.py
🟢 Track B → infra/ или .github/workflows/ (NEW files only)
🟡 Track C → .forgeplan/<kind>/ via MCP only (single MCP track parallel — не concurrent MCP writes)
```

**Checklist перед spawn**:
- [ ] Каждый track touches DIFFERENT files?
- [ ] pyproject.toml dep additions — только в одном track?
- [ ] grid_runner.py SMOKE_MODELS / WEEKLY_MODELS — только один track?
- [ ] Prompt ≤ 2000 tokens? (см. Bad practices)

### 3. Progressive validation — 1×1×1 → full grid

Перед full smoke run ($5-15 spend), всегда запускать **1 task × 1 model × 1 seed = 1 eval ($0.02)**. Validates:
- Pre-flight checks (proxy + auth + model routing)
- Manifest write
- Postmortem generation
- File mode 0o444 (ADR-002 immutability)
- Cost reconciliation

Каждый of 4 bugs в Phase 2B coda был caught at $0.02 stage (saved $5-15 wasted).

### 4. Forgeplan operating contract

- ALL artifact mutations через `mcp__forgeplan__forgeplan_*` MCP tools — **никогда** direct `.md` edits (red line in CLAUDE.md)
- `forgeplan_validate` после каждого `forgeplan_update`
- `forgeplan_link source target relation` — для cross-references
- `forgeplan_score <ID>` — R_eff = min(evidence_scores), weakest-link
- `forgeplan_new(kind, title, parent_id=...)` — parent_id auto-links только для `kind=evidence`. Для adr/rfc/prd/spec/note нужно явный `forgeplan_link` после create.

### 5. NOTE-002 Evidence Quality Standard

Каждый EVID должен иметь:
- `## Structured Fields` (verdict + congruence_level + evidence_type)
- `## ADI cycle` (≥3 hypotheses + deductive predictions + induction table)
- `## Trust Calculus per claim` (F + G + R per claim, ideally sum ≥ 12/27)
- `## Conclusions` (surviving hypothesis + decision strength + follow-up)

### 6. Commit hygiene

- Conventional Commits (`feat:`, `fix:`, `refactor:`, `docs:`, `chore:`, `ci:`)
- Body: explain *why*, не just *what*
- `Refs: <slug>, <slug>` references к artifact IDs
- HEREDOC pattern: `git commit -m "$(cat <<'EOF' ... EOF)"`
- **НИКОГДА** `--no-verify` — lefthook safety net

### 7. Hindsight retain — non-obvious findings

Save в memory только то что:
- Non-obvious из git/code
- Has "Why" + "How to apply" sections
- Outdated info quickly (current PR list, recent commits) — НЕ save

## ❌ Bad practices (избегайте)

### Anti-pattern 1: Re-implementing library APIs

```
❌ "Я напишу свой cyclomatic counter — это 50 строк"  → use radon/lizard
❌ "Свой retry decorator с exp backoff"               → use tenacity
❌ "Кастомный YAML parser"                            → use ruamel.yaml/PyYAML
❌ "Своя impl Krippendorff α"                         → use krippendorff PyPI
❌ "Hardcoded model pricing dict"                     → use litellm.cost_per_token()
```

### Anti-pattern 2: Long agent prompts (>2000 tokens)

В сессии 2026-05-24 один coder упал с `"Prompt is too long"` — потерян час работы. Lesson:
- Spawn prompts ≤ 2000 tokens
- Reference artifacts via `forgeplan_get id=X` instead of inlining body
- Use `Mandatory reading: 1. X 2. Y` instead of pasting full content
- Trust agent to read what it needs (don't duplicate context)

### Anti-pattern 3: Spec drift assumptions

В Slice B coder caught assumption-error в task spec (`eval_result.raw_output` field не существовало). 
**Lesson**: agents должны проверять real signatures (Context7 + actual file read) BEFORE coding, не slепо trust task description.

### Anti-pattern 4: Bogus tests

Determinism check изначально compared real LLM output vs FakeEvalCaller output — это всегда failed by design. Violation of ADR-002 reproduce semantics. **Lesson**: test must match the spec contract, not just "run something".

### Anti-pattern 5: Direct `.forgeplan/**/*.md` edits

RED LINE per CLAUDE.md. Direct sed/Edit/Write desync's LanceDB index + state machine.
**Always** через `mcp__forgeplan__forgeplan_*` или shell `forgeplan` CLI.

### Anti-pattern 6: Concurrent MCP writes на same artifact kind

Per CLAUDE.md, parallel `forgeplan_update` calls on same kind могут collide.
- 2 PRD-mutating tracks parallel = OK (different IDs)
- 2 simultaneous updates of same PRD = sequential

### Anti-pattern 7: Push без PR

`main` merge только через PR. No direct `git push origin main`. Red line per CLAUDE.md.

### Anti-pattern 8: Outdated hardcoded values

Hardcoded gemini-3-flash pricing был 6.7× outdated. LiteLLM `cost_per_token()` caught это автоматически.
**Lesson**: prefer library auto-update over hardcoded snapshots для anything that drifts (prices, schemas, API versions).

## 💡 Insights — что узнали в этой сессии

### Insight 1: Vision Map (NOTE-005) ROI

Один файл = 30 секунд триаж вместо чтения 45 artifacts. Saves hours per session. Pattern: **canonical coverage doc** — must-have для projects > 20 artifacts.

### Insight 2: Context7 = silent safety net

3 immediate wins за сессию без явного involvement:
1. Outdated gemini pricing
2. Outdated gitleaks API
3. RFC pseudocode matrix orientation bug

Без mandatory Context7 step → код был бы deploy с silent bugs.

### Insight 3: Parallel = 3× throughput для ortho work

4 waves в session (3 ortho tracks каждая) → ~12 tasks completed в parallel vs sequential. Key: explicit file ownership pre-spawn.

### Insight 4: R_eff structural cap

PRD-001 R_eff stuck на 0.30 даже после 5 EVIDs at score 1.0. Cause: EPIC-001 parent CL penalty. **Lesson**: добавление evidence НЕ всегда лифтит R_eff — нужно address weakest-link first (supersede weak evidence OR address structural CL penalty).

### Insight 5: Phase numbering — два разных axes

- **Codebase phases** (Phase 0-5) — what to build infrastructure-wise
- **Research phases** (Фаза 1-5 per dd.md) — what to measure experimentally

Не путать. NOTE-004 Section 6 explicit disambiguation.

### Insight 6: Cost realism

Estimated $5-15 для 45-eval smoke — actual $0.067 (25× cheaper). Reasons:
- Outputs ~3.5-4.6k tokens, не 5k+ pessimistic
- Inputs ~30-50 tokens для smoke prompts (minimal)
- Cheap models dominate token count
- Только Claude Sonnet 4.6 ($15/Mtok output) drives 80% cost

### Insight 7: Progressive validation cheap

1×1×1 eval ($0.02) caught 2 bugs which would have wasted $5-15 в full 45-eval blind run. **Always run minimum first**.
