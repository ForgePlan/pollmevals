# Documentation rubric v0.1.0

Score each criterion from 0 to 10.

## Criteria

1. Structural completeness: all expected sections are present.
2. Factual accuracy: no contradictions with the task input or reference.
3. Clarity: a junior developer can understand it.
4. Actionability: the reader can execute the instructions.
5. Consistency: terms, commands and assumptions do not conflict.

Do not score based on length. Long is not automatically better.
