## What this gold solution proves

- Renders three linear steps (personal → address → review) with Next / Back navigation gated by per-blur validation.
- Discriminates UI states via a tagged union `{ kind: "idle" | "loading" | "success" | "error" }` — no impossible state combinations.
- Persists the draft to `sessionStorage` under the stable key `fe01:multistep-form:draft`, hydrates on mount, clears on successful submit.
- Manages focus deterministically: moves to the first focusable of the new step on transition; moves to the first invalid input when forward progress is blocked.
- Announces step transitions and submission outcomes via a polite `aria-live` region (`role="status"`); errors render in `role="alert"` for assertive announcement.
- Binds labels via `htmlFor` / `id` (generated by `useId`), links errors via `aria-describedby` + `aria-invalid="true"` — axe-core reports 0 serious / 0 critical on every step.
- Uses semantic elements (`<form>`, `<fieldset>`, `<legend>`, `<nav>`, real `<button>`) — no `div`-as-button.
- Exports pure helpers (`validateField`, `fieldsForStep`, `isStepValid`, `hydrateDraft`) so hidden tests can exercise them in isolation without driving the UI.
- Sets `autoComplete` per WHATWG section 17.5 (`name`, `email`, `street-address`, `address-level2`, `postal-code`) — credential-manager / browser autofill friendly.

2. **AccessibilityEvaluator**: parses axe-core results from the test JSON output; score = `1 - serious_critical_count / max(violations, 1)` saturating at 0.
3. **TypeSafetyEvaluator** (host-side per NOTE-007): runs `tsc --noEmit --strict --pretty false`. Score = `1 - min(1, errors/10)`.

5. **ComplexityEvaluator**: lizard Python API on the solution AST. Score saturates at CC ≤ 6.
6. **CoverageEvaluator**: reads v8 coverage JSON; score = mean(line, branch, function) coverage.
7. **JudgePanel**: 3 vendor-family judges score 6 rubric criteria (see `../rubric.yaml`). Median per criterion + Krippendorff α + bootstrap CI per RFC-002.
